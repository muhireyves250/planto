# Deploying Planto

This document outlines the steps to deploy the Planto application to production using **Render** (for the backend) and **Vercel** (for the frontend). 

## 1. Commit and Push
First, ensure all your code is committed and pushed to your GitHub repository.
```bash
git add .
git commit -m "Configure deployment and environment variables"
git push origin main
```

## 2. Backend Deployment (Render)

We have configured `render.yaml` which automatically configures the backend environments for Render.

1. Go to [Render's Dashboard](https://dashboard.render.com).
2. Click **New** -> **Blueprint**.
3. Connect your GitHub repository.
4. Render will detect the `render.yaml` file and create the `planto-backend` web service.
5. In the Render Dashboard, go to your new web service's **Environment** tab.
6. Make sure to provide the values for the following un-synced variables:
   - `DATABASE_URL` (Your Neon connection string)
   - `SMTP_EMAIL`
   - `SMTP_PASSWORD`
   - `VITE_OPENWEATHER_API_KEY`
7. Click **Save Changes** and wait for the deployment to finish.
8. Once complete, copy the backend URL (e.g., `https://planto-backend.onrender.com`).

## 3. Frontend Deployment (Vercel)

Vercel provides a seamless experience for Vite-based React apps.

1. Go to [Vercel's Dashboard](https://vercel.com/new).
2. Click **Add New...** -> **Project**.
3. Import your GitHub repository.
4. When configuring the project:
   - **Framework Preset**: Vite
   - **Root Directory**: Select `/frontend`
5. In the **Environment Variables** section, add:
   - Name: `VITE_API_URL`
   - Value: The Render URL you copied in the previous step (e.g., `https://planto-backend.onrender.com`).
6. Click **Deploy**.

## Verification
- Visit the URL generated by Vercel for your frontend domain.
- Attempt to log in or create an account to ensure the backend is connected and the API requests are succeeding.

> [!TIP]
> **CORS Policy:** If you encounter CORS issues after deploying, verify that `VITE_API_URL` does NOT end with a trailing slash in your Vercel environment variables, and make sure FastAPI is configured to allow your Vercel domain in its CORS middleware (typically located in `backend/app/main.py`).
